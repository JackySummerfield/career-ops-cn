# Workflow 5: 面试复盘

触发词："面试复盘""面试录音""复盘面试""debrief""面试反馈""录音整理"。

## Input

用户提供面试录音音频文件路径，或已转录的面试文字记录。可选附加：职位信息（tracker id 或公司+角色）。

## Steps

### Stage A: 音频转录与结构化

1. **转录** — 如输入是音频文件，调用 `everything-to-md` skill 的音频转录能力将录音转为文字。
2. **话者分离** — 识别面试官 vs 候选人发言（基于上下文、提问/回答模式、称呼等线索）。
3. **分段** — 按问答对切分，每个单元包含：
   - 面试官提问（原文）
   - 候选人回答（原文）
   - 问题分类标签

### Stage B: 逐题分析

对每个问答对输出：

| 字段 | 说明 |
|------|------|
| 问题分类 | 行为/技术/动机/压力/追问 |
| 问题意图 | 面试官想考察什么 |
| 回答摘要 | 候选人实际回答的核心内容（3-5句） |
| 回答质量 | ⭐~⭐⭐⭐⭐⭐ + 一句话评价 |
| 亮点 | 回答中做得好的部分 |
| 改进建议 | 具体可操作的改进方向 |
| 推荐答法 | 如果重新回答，建议的结构和要点 |

### Stage C: 总结与模式识别

1. **整体表现评估** — 沟通节奏、结构化程度、自信度、故事完整性
2. **高频问题识别** — 与该用户历次面试复盘交叉比对（如有多次复盘记录），识别反复被问的问题类型
3. **回答模式问题** — 识别系统性弱点（如：总是缺数字、STAR 结构不完整、容易跑题）

### Stage D: 反哺母版简历

基于复盘发现，向用户建议以下更新（需用户确认后执行）：

1. **新增标准化故事** — 高频问题 + 本次回答中的好素材 → 建议形成 STAR 故事写入 `cv_master.md` "三 → B. 个人故事"
2. **优化现有故事** — 本次面试中发现更好的表达/数据 → 建议更新已有故事
3. **补充事实** — 面试中提到但母版尚未记录的项目细节、数据、成果 → 建议补充
4. **沉淀表达策略** — 经用户确认的自我介绍、动机题和标准答法 → 建议写入 `interview_playbook.md`
5. **标注风险** — 面试中被追问暴露的薄弱点 → 建议标注到求职偏好/真实性边界

> 执行母版更新时遵循 w0.5 的 STAR 结构化流程和中英对照规则。

### Stage E: 更新 timeline.md

在 `<USER>/jobs/{id:03d}_{company}_{role}/timeline.md` 追加面试记录：
- 日期、轮次、面试官角色（如可判断）、核心问题列表、自评总结

## 存储

```
<USER>/jobs/{id:03d}_{company}_{role}/
├── interview_debrief_r1.md   # 第1轮复盘（r1=round 1）
├── interview_debrief_r2.md   # 第2轮复盘（按轮次递增）
└── timeline.md               # 自动追加记录
```
