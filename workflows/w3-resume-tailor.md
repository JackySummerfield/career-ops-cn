# Workflow 3: 简历版本与JD轻量适配

触发词："生成目标版本简历""根据母版推荐投递方向""生成某方向稳定简历""针对#6给我定制建议""基于某方向版本适配这个JD""轻量定制简历""采纳建议"。

## Core Principles

1. `<USER>/resume/cv_master.md` 是母版事实库，不直接投递，不因单个JD被覆盖。
2. 稳定投递版本按用户确认的投递方向生成，保存到 `<USER>/resume/versions/{direction}_cn.md` / `{direction}_en.md`。
3. 具体JD层只先生成轻量适配建议，默认不直接生成最终简历。
4. 用户明确确认采纳哪些建议后，才生成或更新 `<USER>/jobs/{company}_{role}/cv_cn.md` / `cv_en.md`。
5. **绝不添加母版简历中没有的经历、技能、结果、团队规模、用户规模、技术栈、产品权限或管理人数**。
6. 允许的改动仅限：排序、删减、同义词替换、基于真实事实的表达压缩/展开、将事实映射到JD关键词。
7. 不写"求职意向"段落，除非用户明确要求。

## Stage A: 投递方向诊断（如尚未完成）

当用户还没有明确方向，或初始化后尚未生成 `<USER>/resume/direction_diagnosis.md` 时：

1. 读取 `<USER>/resume/cv_master.md`。
2. 基于真实经历推荐投递方向，不预设固定方向。
3. 输出并保存 `<USER>/resume/direction_diagnosis.md`，格式如下：

```markdown
# 投递方向诊断

| 方向 | 优先级 | 推荐理由 | 适配岗位样例 | 核心卖点 | 主要Gap | 风险/真实性边界 | 是否建议生成稳定版本 |
|------|--------|----------|--------------|----------|---------|------------------|------------------------|
| ... | 主攻/过渡/保底 | ... | ... | ... | ... | ... | 是/否 |

## 用户确认结果
- 采纳方向：...
- 合并/重命名：...
- 暂不生成：...
```

4. 让用户确认采纳、合并、重命名或放弃哪些方向，并标记主攻/过渡/保底。

## Stage B: 生成稳定投递版本

当用户确认一个或多个投递方向后：

1. 读取母版简历、direction_diagnosis。
2. 为每个确认方向生成稳定版本，不针对单个JD大改。
3. 每个版本必须包含：
   - 文件最顶部的说明表（非投递正文）：版本定位、适用岗位、不适用岗位、使用方式、面试风险提示
   - 表格下方的简历标题与正文
   - 工作经历
   - 项目经历
   - 技能关键词
4. 保存到 `<USER>/resume/versions/{direction}_cn.md` / `{direction}_en.md`。

## Stage C: JD轻量适配建议

当用户针对某个tracker职位或JD要求"定制简历"时：

1. 先选择最合适的稳定投递版本；如无法判断，向用户确认使用哪个版本。
2. 读取目标版本、`<USER>/jobs/{company}_{role}/eval.md`（或JD原文）、tracker信息。
3. 只输出适配建议，保存到 `<USER>/jobs/{company}_{role}/cv_suggestions.md`，不直接改最终简历。
4. 建议类型固定为：
   - `排序调整`
   - `关键词同义替换`
   - `项目删减/保留`
   - `表达强化`
   - `风险表述`
5. 风险等级固定为：
   - `低风险-事实表达`
   - `中风险-合理抽象`
   - `高风险-需用户确认真实性`
6. 建议清单格式：

```markdown
# 简历轻量适配建议：[公司] [职位]

## 匹配摘要
- 使用基础版本：`resume/versions/{direction}_cn.md`
- JD核心关键词：...
- 当前版本已覆盖：...
- 建议补强：...

## 建议清单

| ID | 类型 | 风险 | 原文/位置 | 建议改法 | 理由 | 是否采纳 |
|----|------|------|-----------|----------|------|----------|
| 1 | 关键词同义替换 | 低风险-事实表达 | ... | ... | ... | 待确认 |

## 禁止改写项
- 未在母版中出现的经历、成果、团队规模、技术栈、产品权限，不得新增。
```

## Stage D: 用户确认 & 生成最终简历

用户说"采纳建议"或逐条确认后：

1. 基于稳定版本 + 采纳的建议，生成最终简历
2. 保存到 `<USER>/jobs/{company}_{role}/cv_cn.md` / `cv_en.md`
3. 更新 `cv_suggestions.md` 中"是否采纳"列
