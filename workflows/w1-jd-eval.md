# Workflow 1: JD评估

## Input

用户提供：JD 文本、URL、截图或附件。

## Steps

1. **解析JD** — 文本直接解析；URL 或截图必须按 [references/jd-fetching.md](../references/jd-fetching.md) 的分层策略处理。Codex 在静态读取不完整时，先尝试可用的浏览器控制与搜索摘要；页面明确阻断自动化时停止重试，接受截图并直接视觉识别，不要求用户额外做 OCR。保存来源 URL，并在评估前检查职位名称、公司、职责和要求是否齐全。
2. **读取简历与偏好** — 读取 `<USER>/resume/cv_master.md`，了解用户背景和求职核心目标。
3. **10维度评分** — 输出标准表格：

| 维度 | 分数 | 说明 |
|------|------|------|
| 角色匹配 | x.x | JD核心职责与经验重合度 |
| 技能覆盖 | x.x | 必备技能覆盖率 |
| 经验年限 | x.x | 年限匹配度 |
| 行业契合 | x.x | 行业背景加分 |
| 成长空间 | x.x | 对职业发展帮助 |
| 薪资预估 | x.x | 与预期匹配度 |
| 公司规模 | x.x | 体量与稳定性 |
| 地理位置 | x.x | 通勤/搬迁接受度 |
| 文化契合 | x.x | 工作风格 |
| 竞争难度 | x.x | 预估竞争激烈程度 |

4. **综合评分** — 加权平均，附一句话建议（≥3.5建议申请，<3.0建议跳过）
5. **Gaps & 亮点** — 列出关键gap和你的差异化优势
6. **薪资区间** — 如JD提供，标注；否则根据市场估算

## Output format

```markdown
## 评估：[公司] [职位]（[城市]）

### 10维度评分
[表格]

### 综合评分：X.X / 5.0
[一句话建议]

### 关键Gap
- ...

### 差异化亮点
- ...
```
